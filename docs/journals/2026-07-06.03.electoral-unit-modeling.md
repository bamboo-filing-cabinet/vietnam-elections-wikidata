# Modeling Vietnamese NA electoral units on Wikidata

Decided 2026-07-06. **Reverses** the "do not create constituency items" stance
in `2026-07-04.03.entity-and-constituency-modeling.md` — see amendment there.

## Why we now create them

Pointing `P768` (electoral district) at the province (Q26575) is imprecise: a
province holds several đơn vị bầu cử and a delegate wins only one. No existing
Wikidata item matches a unit (its area spans multiple districts, so no single
geographic item is both precise and correct), and none exist for VN NA units.
Multi-member systems that record districts (e.g. Malta, instance of *electoral
district of Malta*) point P768 at a dedicated unit item. We do the same.

## Key facts

- Class `Q192611` (**electoral unit**; aliases: constituency, electoral
  district…) is an allowed `P768` value type.
- VN units are **per-election** (redrawn each cycle), unlike Malta's stable
  districts. So each item is **cycle-scoped**: a "No. 2" unit in 2021 is a
  *different item* from a "No. 2" unit in 2026.

## Evidence that units carry no cross-election continuity (primary source)

The 2026 (NA16) official candidate list (chinhphu.vn) vs. our 2021 (NA15) data,
for Thái Nguyên province:

| | 2021 (NA15) | 2026 (NA16) |
| --- | --- | --- |
| Đồng Hỷ + Võ Nhai are in | **electoral unit No. 2** (+ Thái Nguyên city), 3 seats | **electoral unit No. 1** (+ ~a dozen wards/communes), 2 seats |
| Component type | districts (huyện) | wards (phường) + communes (xã) |

The same areas moved from unit No. 2 to unit No. 1, with different partners and
a different seat count — so unit numbers do **not** track a persistent entity
across elections. This is why we scope each unit item to its cycle (and it
contrasts with US congressional districts, which *are* continuing institutions
redistricted on a schedule → single persistent item). The component-type change
(districts → wards/communes) reflects the 2025 abolition of the district tier;
a 2026 unit item's P527 parts would be wards/communes, not districts.
- Source data: `vietnam-elections` `constituencies.json` has all 186 NA15 units
  with `unit_number`, `seat_count`, and component `districts[]`.

## Item pattern (one per cycle × province × unit_number)

**Labels** (place-first, matching Wikidata convention for numbered districts —
e.g. "Arkansas's 4th congressional district", "Bataan's 1st congressional
district"; parenthetical disambiguator because VN units are per-election)
- en: `{province} electoral unit No. {n} ({ordinal} National Assembly of Vietnam)`
- vi: `Đơn vị bầu cử số {n} tỉnh/thành phố {province} (Quốc hội khóa {roman})`

Example en: `Thái Nguyên electoral unit No. 2 (15th National Assembly of Vietnam)`.
Term "electoral unit" matches the class (Q192611) and the source term *đơn vị
bầu cử*. No possessive ("Thái Nguyên's") — awkward on VN place names.

**Descriptions** encode cycle + composition + seats, e.g. en: "multi-member
electoral district for the 2021 Vietnamese legislative election (15th National
Assembly); {seats} seats; comprises …".

**Statements**
| Property | Value | From |
| --- | --- | --- |
| P31 instance of | Q192611 (electoral unit) | fixed |
| P17 country | Q881 (Vietnam) | fixed |
| P131 located in the administrative territorial entity | province item | `locality_id` → QID |
| P1342 number of seats | `seat_count` | data |
| P527 has part(s) | one per component district QID | `districts[]` → QIDs |

Use **P527 (has part(s))**, not P150 (contains the administrative territorial
entity). P150 is the strict inverse of P131, so it would falsely imply each
component district is administratively located *in the electoral unit* — but the
districts are administratively in the province. An electoral unit only *groups*
them for one election, which is exactly what P527 expresses.

Reference all statements to the source in our data for unit composition:
"Congressional units (PDF)" (`doc-550cd8ad955d`), reference URL
`https://images.hcmcpv.org.vn/Uploads/File/280420219523F244/Danhsachbaucu-PYFO.pdf`,
retrieved 2026-01-02.

Then on each delegate: set the P39 qualifier `P768` to the unit item (not the
province).
