# NA15 reconciliation run (Phase 1)

Ran `src/reconcile.py na15-2021` on 2026-07-06 against live Wikidata.
Output: `reconciliation/na15-2021.csv` (500 rows, one per winning delegate).

## Results

| match_status | count | meaning |
| --- | --- | --- |
| matched | 204 | exactly one Wikidata item matches on folded name **and** day-precision DOB |
| ambiguous | 218 | name matches but DOB could not confirm it (see below) — needs human review |
| unmatched | 78 | no name candidate on Wikidata — creation candidates |

## How matching works

Name-first, DOB-confirmed (Vietnamese names collide heavily, so a name match
alone is never treated as confident):

1. Pull all current holders of `P39 = Q10841192` (member of the NA of Vietnam)
   with folded names + DOB in one SPARQL query (~1,460 name→QID entries).
2. Winner's folded name (diacritics stripped, `đ`→`d`) hits the pool → confirm
   by DOB. Exactly one DOB match → `matched`; else `ambiguous`.
3. No pool hit → search Wikidata by name (vi + folded en), fetch candidates,
   confirm by DOB.

## Why "ambiguous" is the biggest bucket

Most ambiguous rows are a **name match whose DOB we could not confirm**, not a
genuine name collision. Common causes, both real in this run:

- **Placeholder DOB on Wikidata** — many items store a year-precision date that
  renders as `YYYY-01-01`. Example: Phan Văn Giang (`Q20026119`) is stored as
  1960-01-01; our data has 1960-10-14. The matcher only accepts **day-precision**
  P569, so it declines to auto-match — correctly.
- **Off-by-one / conflicting DOB** — e.g. Nguyễn Tân Cương (`Q20025996`): our
  data 1966-02-12 vs Wikidata 1966-02-11. Flagged for review rather than merged.

So many ambiguous rows are probably correct matches that just need a human to
verify the DOB (and fix whichever side is wrong). The `match_notes` column
carries the candidate QID(s) to check.

## Caveats

- The matcher is deliberately conservative: it prefers `ambiguous` over a
  false-positive `matched`. Expect the true match rate to rise after manual
  review of the ambiguous bucket.
- `matched` QIDs are name + exact-DOB; still worth a sample audit before any
  backfill, since a shared name + shared birthday is not impossible.
- Re-running resumes: `reconcile.py` skips `person_id`s already in the CSV, so
  delete rows (or the file) to force a re-check.
